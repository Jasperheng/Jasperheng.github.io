30. 串联所有单词的子串

一开始不会做

（1）因为不在意子串中各个词组的排列关系，因此将其进行hash map，然后判断其和给定的字符串数组对应的hash map进行对比，若相同则满足条件。（2）每次移动并非移动1格，而是一个单词的长度，所以就只要erase出去的单词，然后加入进来的单词即可，不用重新统计整个长窗口内的hash map，这样也导致要用多个起点（因为每次移动单词个长度会使得一些index被漏掉）。

代码中注意点：for(int j=i; j+m*l<=n; j+=l){…}

102. 二叉树的层序遍历

一开始不会做，忘了二叉树和BFS相关知识，再加上这题需要同时将同一层的value放在一起

不是普通的BFS，附加了输出条件。方法1：每次记录层数（上层+1），然后用层数作为hash map的key，最后按key输出；方法2：每次不是pop一个结点出来，而是pop当前的所有结点出来（pop出来的就是同一层的所有结点）

相关知识：

BFS与DFS

  • BFS（广度优先搜索），也叫层序遍历

    使用队列（queue）来实现，它按层次从上到下、从左到右遍历树的每一层。

    实现思路（二叉树’s Version）：

    1. 从根节点开始，将其加入队列。

    2. 从队列中取出节点，访问该节点，并将其左右子节点按顺序加入队列。

    3. 重复以上步骤，直到队列为空。

  • DFS（深度优先搜索）

    深度优先遍历有三种主要方法：前序遍历、中序遍历和后序遍历。它们都可以通过递归或使用栈（stack）来实现。X序代表根节点在什么时候被访问。

    实现思路：

    1. DFS方法首先从根节点开始，将根节点入栈

    2. 从stack中访问栈顶的结点

    3. 找出与此点邻接的且尚未遍历的点，进行标记，然后放入stack中，依次进行；

    4. 如果此点没有尚未遍历的邻接点，则将此点从stack中弹出，再按照3依次进行；

    5. 直到遍历完整个树/图，stack里的元素都将弹出，最后栈为空，DFS遍历完成（‌树和图的主要区别在于树是无环的连通图，而图可以有环）

130. 被围绕的区域

这题要注意一些代码小细节，如最后一行是>=(m-1)+n；按我的写法if(p-n>=0 && board[(p-n)/n][(p-n)%n]==’O’ &&!visited[(p-n)/n][(p-n)%n]){…}这里要重新计算横纵坐标，而不是用i-1和j，因为i,j对应的是起始元素的坐标，而现在已经在遍历连通图了。

530. 二叉搜索树的最小绝对差

一开始不会做（不记得二叉搜索树的定义及性质）

定义：二叉查找树(Binary Search Tree)，(又:二叉搜索树，二叉排序树)它或者是一棵空树，或者是具有下列性质的二叉树: 若它的左子树不空，则左子树上所有结点的值均小于它的根结点的值; 若它的右子树不空，则右子树上所有结点的值均大于它的根结点的值; 它的左、右子树也分别为二叉排序树。

性质：中序遍历后结果为递增序列

因此，这题可以用一个序列存储中序遍历的结果，然后再计算相邻位置的最小差值，或者在遍历的过程中用一个pre变量存储先导元素，然后不断更新结果。

222. 完全二叉树的节点个数

这题遍历计算总数就行了，但是进阶版：遍历树来统计节点是一种时间复杂度为 O(n) 的简单解决方案。你可以设计一个更快的算法吗？

再看一遍题解！主要是在最后一层进行二分查找+位运算，需要利用完全二叉树的性质，在整数和二进制数之间转换

相关知识：

  1. 在完全二叉树内，从根结点到某一叶子结点，若走左子树则对应位为0，右子树则为1，这样最终构成的二进制序列转换为十进制后，是从根节点层序遍历到当前叶子结点的总结点个数。

  2. 在完全二叉树内，假设一共h层（0,1,…,h），最后一层的结点个数在[$1,2^h$]之间，对应的整棵树的总结点数量在[$2^h,2^{h+1}-1$]之间。

  3. 等比数列求和公式:

    $(q\neq1)S_n=a_1(1-q^n)/(1-q)=(a_1-a_nq)/(1-q)$

    $(q=1)S_n=na_1$

  4. a&b: 按位与操作，如果对应位置都是1，则结果为1，否则为0。

128. 最长连续序列

一开始记错快排时间复杂度（平均O(nlogn)，最坏即有序情况O(n^2)），直接用快排然后再判断，但应该是：每个数都判断一次这个数是不是连续序列的开头那个数。用哈希表查找这个数前面一个数是否存在。存在则直接跳过；不存在则往后一直查找到没有，然后记录并更新最大长度，这样的时间复杂度就可以达到题目要求的O(n)。还有方法是把每个数向前和向后找（找到的元素删掉）。
